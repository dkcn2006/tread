# tread - Code Plan

## 项目概述

**tread** 是一个伪装性极强的终端 TUI 小说阅读器，用 Rust 实现。核心设计目标：只占终端 1-3 行，看起来像普通的终端输出（日志 / 纯文本 / 代码注释），适合摸鱼场景。

## 模块职责

| 文件 | 职责 |
|------|------|
| `main.rs` | CLI 参数解析（clap）、终端初始化（Inline viewport）、panic 恢复、退出清理 |
| `app.rs` | 应用状态机（Normal/Search/ChapterList）、事件循环、键盘映射、书签保存 |
| `reader.rs` | 文件读取、编码检测、章节解析、智能折行（unicode-width）、全文搜索 |
| `ui.rs` | 三种伪装模式渲染（Log/Minimal/Comment）、搜索覆盖层、章节目录、状态提示 |
| `bookmark.rs` | JSON 书签持久化到 `~/.config/terminal-read/bookmarks.json` |
| `config.rs` | `DisplayMode` 枚举定义及切换逻辑 |

## 关键设计决策

| 决策 | 选择 | 原因 |
|------|------|------|
| Viewport 模式 | Inline（非 AlternateScreen） | 隐蔽性：看起来像普通命令输出 |
| 编码检测策略 | 依次尝试 UTF-8 → BOM → GBK → GB18030 → BIG5 | 覆盖绝大多数中文 txt 小说 |
| 字符宽度计算 | `unicode-width` crate | 正确处理中文占 2 列宽 |
| 书签 key | 文件 canonical 路径 | 避免相对路径导致的书签匹配失败 |
| 事件轮询 | 250ms timeout | 平衡响应速度和 CPU 占用，日志时间戳可更新 |
| 章节识别 | 正则 + 宽度过滤 (≤60) | 避免正文中碰巧包含"第X章"的长句被误判 |
| 逐显示行滚动 | 基于 sub_offset 的子行级导航 | 长段落折行后按 j/k 不会跳过内容 |
| 老板键 | Esc 清屏 + 光标归零 | 一键消失，不留痕迹 |
| 状态反馈 | `status_message` + 2 秒自动过期 | 搜索失败 / 无章节等场景轻量提示 |

## 测试覆盖

单元测试位于各模块的 `#[cfg(test)] mod tests` 中：

- `reader::tests` — `wrap_line`（空行/ASCII/中文/混合/恰好 fit）、`search_forward`（基础/回绕/无匹配/空书）、`parse_chapters`（中文/英文/跳过超长行/宽度过滤/卷篇）
- `bookmark::tests` — 序列化反序列化 roundtrip、缺失 key、覆盖写入、JSON 格式验证
